# Release Notes — AI Code Factory v3.4.0 Orchestrator Brain Edition

## Headline

`ask` is now an intelligent orchestration step.

v3.4 adds an AI Intake Brain capable of understanding intent, difficulty, scope, risk, missing information, decomposition needs, design-first requirements and next actions before any executor touches code.

## New core features

### AI Intake Brain

New modules:

- `src/engines/ai-intake-brain.js`
- `src/engines/ai-intake-provider.js`
- `src/engines/intake-schema.js`

Behavior:

```text
ask
→ heuristic pre-analysis
→ AI provider when configured
→ strict JSON schema validation
→ deterministic safety merge
→ fallback when unavailable/invalid/low confidence
→ `.ai` artifacts
```

### Difficulty and scope

The intake result now includes:

- `difficulty`: `trivial`, `simple`, `medium`, `complex`, `epic`
- `scope`: `single_file`, `single_feature`, `multi_file`, `fullstack_slice`, `product_epic`
- `should_implement_now`
- `requires_decomposition`
- `next_best_action`

### Product epic handling

Large asks like “build a Vinted-like marketplace” are classified as `product_epic`, not a single implementation REQ.

New module:

- `src/engines/epic-decomposer.js`

Artifacts:

- `.ai/epics/EPIC-001.json`
- `.ai/epics/EPIC-001-roadmap.md`
- `.ai/epics/index.json`

### Brain artifacts

New files per REQ:

- `.ai/reasoning/brain/REQ-XXX-brain-summary.md`
- `.ai/reasoning/brain/REQ-XXX-decision-log.md`
- `.ai/reasoning/brain/REQ-XXX-brain-context.json`

### API key config

New config surfaces:

- `.env.example`
- `.ai/config.local.example.json` generated by `init`
- `npm run ai -- brain-status`

### Context pack upgrade

Context packs now include:

- Orchestrator Brain decision;
- user answers/clarifications;
- project DNA;
- learned rules;
- compiled memory;
- design and engineering taste;
- locked constraints;
- suggested roadmap/REQ slices;
- allowed files strategy.

### Hard gate fixes

- `approve --dry-run` no longer closes work as `done`.
- executor missing/skipped/failed/timed out blocks close.
- product epics cannot be implemented as one huge REQ unless explicitly forced.
- new scope gate compares touched files against the allowed contract.

### Dashboard test hardening

`startDashboard(root, { port: 0 })` now respects port `0`; this avoids test/runtime port collisions.

## Migration from v3.3.1

1. Replace the harness folder with v3.4.
2. Run:

```powershell
npm run ai -- init
npm run ai -- brain-status
```

3. Add your API key via `.env` or environment variable.
4. Test with:

```powershell
npm run ai -- ask "Quiero una app tipo Vinted para tesis con usuarios, publicaciones, ofertas, chat y pagos simulados"
npm run ai -- why
npm run ai -- context-pack
```

## Honest limitation

The real OpenAI call requires a valid API key and API billing/access. Without it, v3.4 intentionally falls back to deterministic intake instead of blocking your workflow.
